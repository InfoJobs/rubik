Rubik

Rubik es la guia front-end del portal de empleo InfoJobs. Aquí se presentan todos los elementos, componentes y elementos Javascript que se usan en el portal

El objetivo de la guia es doble: estandarizar y reutilizar elementos y componentes, por una parte y hacer más autónomo al equipo de desarrollo por otra.

¿En qué nos basamos?

Para la construcción de la guia nos hemos aprovechado de cuatro pilares básicos.

OOCSS

Oriented Object CSS tiene como objetivo principal agregar nuevos comportamientos a un elemeto escribiendo el mínimo código posible.

Por ejemplo: tenemos distintos mensajes como resultado de una acción. Si separamos estructura de presentación, podemos cambiar la presentación sin necesidad de repetir todo el código cada vez. La base es siempre la misma, pero será de un color o otro dependiendo del caso.

Mobile first & Responsive design

Mobile first nos ayuda a centramos en lo que es realmente importante, jerarquizando y estructurando el contenido.

Lo que no es importante para el móvil, continua siendo superfluo para el desktop.

El responsive design nos aporta escalabilidad. De esta manera tenemos una correcta visualización y experiencia en las distintas resoluciones y dispositivos.

Sass

Syntactically Awesome Stylesheets nos ayuda a escribir y estructurar hojas de estilo de forma más limpia y controlada.

Gracias a Sass, passamos de una sóla hoja de estilos CSS con centenares de lineas, a una serie de hojas de estilo parciales, que usan variables, mixins, anidaciones, etc.

El resultado acaba siendo también una sóla hoja de estilos CSS, pero ésta es más corta y mejor estructurada, ya que sólo contiene lo necessario para cada página.

Javascript

Si bien en InfoJobs nos basamos en CSS3 para realizar animaciones y transiciones, no siempre todas las necesidades las podemos cubrir únicamente con estilos.

Javascript nos ayuda a enriquecer la interfaz y implementar comportamientos como por ejemplo las ventanas modales.
